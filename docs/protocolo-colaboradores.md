# 📋 Protocolo de Relacionamento com Colaboradores

> Versão: 1.0 — 28/06/2026

## 1. Quem é Quem

### Staff de Modelagem e Escrita ("a Cozinha")

| Pessoa | Função | O que faz |
|---|---|---|
| **Fabio Takwara** | Modelador / Diretor de Criação | Estratégia, direcionamento, aprovação, integração com parceiros |

| Papel | Função | O que fornece |
|---|---|---|
| **Pesquisador Responsável (PR)** | Coordena o projeto, submete a proposta | Conhecimento técnico-científico, dados, rede de contatos |
| **Pesquisador Associado (PA)** | Participa da equipe executora | Artigos, dados de campo, revisão |
| **OSP** | Representa a comunidade | Diagnóstico territorial, demandas reais, anuência |

## 2. Como Enviar Material para a Cozinha

### ✅ Faça assim:

1. **Identifique-se** no início de cada áudio/vídeo: "Aqui é [nome], sobre o projeto Desafios da Amazônia..."
2. **Nomeie o projeto** — sempre mencionar "Desafios da Amazônia" ou "UnB + Takwara"
3. **Um assunto por vez** — áudios separados para temas diferentes (orçamento, metodologia, cronograma)
4. **Documentos de referência** — envie PDF/link + uma frase do que espera extrair
5. **Dados sensíveis** — marque como "restrito" para não irem para versão pública

### ❌ Evite:

- Áudios longos misturando 3 projetos diferentes (ex: "sobre o Desafios e também sobre o ECOSALA...")
- Documentos sem contexto ("faz o que tiver que fazer com isso")
- Pular a identificação — áudios podem se misturar na pasta de triagem

## 3. Metodologia de Trabalho

### Fase 1 — Triagem
Colaborador envia material → Cozinha processa (transcreve, extrai, organiza) → Salva em TRIAGEM-BRUTA/

### Fase 2 — Estruturação
Cozinha transforma em documento .md → Organiza na pasta correta → Commit + push

### Fase 3 — Revisão
Disponibiliza link → Colaborador revisa → Cozinha ajusta

### Fase 4 — Finalização
Proposta pronta → Submissão no SIGCONFAP (pelo PR)

## 4. Salvaguardas (CANCU)

**Sem a observância das Salvaguardas de Cancún (REDD+), não se inicia trabalho de campo ou coleta de dados com comunidades — apenas rascunhos internos.**

Isso inclui:
- Consentimento Prévio, Livre e Informado (CPLI)
- Proteção de conhecimentos tradicionais
- Soberania de dados territoriais
- Transparência na repartição de ativos

Protocolo de referência: MQTF — `GOV_PROTOCOLO_SEGURANCA_CANCUN.md`
